Types of Keys in a Database

In a database, keys are used to identify records uniquely and establish relationships between tables. They ensure data consistency, avoid duplication, and help in organizing data efficiently.

1. Primary Key

A Primary Key uniquely identifies each record in a table. It cannot contain NULL values and must be unique for every row.

Example: In a Students table, Student_ID can be the primary key because each student has a unique ID.

2. Foreign Key

A Foreign Key is a column that links one table to another. It ensures that the value in the column matches a value in the primary key of another table.

Example: In an Enrollments table, Student_ID can be a foreign key referencing Student_ID in the Students table.

3. Candidate Key

A Candidate Key is a column, or a set of columns, that can uniquely identify a record. A table can have multiple candidate keys, but one of them is chosen as the primary key.

Example: In a Students table, both Student_ID and Email could uniquely identify a student. Either can be a candidate key.

4. Composite Key

A Composite Key consists of two or more columns combined to create a unique identifier for a record.

Example: In an Enrollments table, Student_ID + Course_ID can form a composite key to uniquely identify each enrollment.

5. Super Key

A Super Key is any set of columns that uniquely identifies a record in a table. All primary keys are super keys, but not all super keys are primary keys.

6. Alternate Key

An Alternate Key is a candidate key that was not chosen as the primary key. It can still uniquely identify records.

Why Keys Are Important

  • Maintain data integrity by avoiding duplicates.
  • Establish relationships between tables.
  • Help retrieve data efficiently using queries.

Example

Example: Consider a school database:

  • Students Table: Student_ID (Primary Key), Name, Email (Candidate Key)
  • Courses Table: Course_ID (Primary Key), Course_Name
  • Enrollments Table: Enrollment_ID (Primary Key), Student_ID (Foreign Key), Course_ID (Foreign Key), Student_ID + Course_ID (Composite Key)

Tip for Beginners

Always define keys carefully while designing a database. Primary keys and foreign keys are essential for maintaining data integrity and building relationships between tables.
