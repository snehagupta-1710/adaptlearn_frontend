Tables, Rows, and SQL Commands – Complete Guide

In a relational database, all data is stored in tables. A table is like a structured spreadsheet where each row represents a record (an instance of data) and each column represents a field (type of data). SQL (Structured Query Language) is used to interact with these tables to store, update, delete, and retrieve data.

1. What is a Table?

A table is a collection of related data organized in rows and columns. Think of it as a classroom: the table is the classroom, columns are the attributes of each student (Name, Age, Marks), and each row is one student sitting in that class.

  • Column: Describes what kind of data is stored (Name, Age, Marks).
  • Row: Stores one record of data (one student’s details).

Example: Students table:

Student_ID Name Class Marks
1 Sneha 10A 95
2 Tanvi 10B 88

2. Columns (Fields)

Columns define the type of data that will be stored in the table. They have a data type which specifies whether the column stores numbers, text, dates, etc.

  • Student_ID: INT (Integer) – unique identifier.
  • Name: VARCHAR(50) – stores text.
  • Class: VARCHAR(10) – class name.
  • Marks: INT – student's score.

3. Rows (Records)

Each row represents a single record in the table. Rows contain actual data for each column.

Example: The row 1, Sneha, 10A, 95 represents Sneha’s record in the Students table.

4. SQL Commands

SQL commands are instructions to manage and manipulate data in tables. They are divided into categories:

  • Data Definition Language (DDL): CREATE, ALTER, DROP (structure of tables).
  • Data Manipulation Language (DML): INSERT, UPDATE, DELETE, SELECT (manipulate data in tables).
  • Data Control Language (DCL): GRANT, REVOKE (control access to tables).
  • Transaction Control Language (TCL): COMMIT, ROLLBACK, SAVEPOINT (control transactions).

5. Step-by-Step SQL Examples

Creating a Table

CREATE TABLE Students (
  Student_ID INT PRIMARY KEY,
  Name VARCHAR(50),
  Class VARCHAR(10),
  Marks INT
);

Explanation: Creates a table named Students with 4 columns. PRIMARY KEY ensures each Student_ID is unique.

Inserting Rows

INSERT INTO Students VALUES (1, 'Sneha', '10A', 95);
INSERT INTO Students VALUES (2, 'Tanvi', '10B', 88);

Explanation: Adds two students’ records to the table.

Updating Data

UPDATE Students SET Marks = 90 WHERE Student_ID = 2;

Explanation: Changes Tanvi’s Marks from 88 to 90.

Deleting Rows

DELETE FROM Students WHERE Student_ID = 1;

Explanation: Removes Sneha’s record from the table.

Selecting Data

SELECT * FROM Students;
SELECT Name, Marks FROM Students WHERE Marks > 85;

Explanation: Fetches data. First query fetches all columns; second query fetches only Name and Marks for students with Marks greater than 85.

6. Common Mistakes Beginners Make

  • Forgetting to define a primary key.
  • Inserting wrong data types (text in INT column).
  • Updating or deleting without a WHERE clause (affects all rows).
  • Not checking results with SELECT after each operation.

7. Tips for Practice

  • Create tables with 3-4 columns and insert 5-10 rows.
  • Practice SELECT with filtering, ordering, and limiting rows.
  • Try updating and deleting specific rows safely.
  • Use small test tables before working on real data.

8. Mini Exercises

  • Create a table called Teachers with columns: Teacher_ID, Name, Subject, Experience.
  • Insert 3 rows of sample teachers.
  • Update one teacher’s experience.
  • Delete a teacher’s record.
  • Write a SELECT query to fetch all teachers teaching a specific subject.
