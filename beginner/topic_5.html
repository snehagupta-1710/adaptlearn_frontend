SQL Commands

SQL (Structured Query Language) is the language used to communicate with databases. Think of a database as a digital notebook or a smart filing cabinet. SQL lets you write instructions to: create, read, update, or delete information. These instructions are called SQL commands.

For example, imagine you run a school and want to manage student information. You need to add new students, update their class, see who is enrolled, or remove students who left. SQL commands help you do all of this easily and quickly.

Categories of SQL Commands

1. Data Definition Language (DDL)

DDL commands help you define the structure of your database. Think of this as designing your notebook: creating pages (tables), adding columns, or removing old pages.

  • CREATE: Make a new table or database.
  • ALTER: Change a table, like adding a new column for phone numbers or changing column names.
  • DROP: Delete a table or database completely. Use carefully, data is lost forever!
  • TRUNCATE: Remove all data in a table but keep the table structure.

Example: Creating a table for students:

CREATE TABLE Students (
  Student_ID INT PRIMARY KEY,
  Name VARCHAR(50),
  Class VARCHAR(10)
);

Here, Student_ID is unique for each student. Think of it like a student roll number. Name stores the student’s name, and Class stores which class the student is in.

Example: Adding a new column for email:

ALTER TABLE Students ADD Email VARCHAR(50);

2. Data Manipulation Language (DML)

DML commands let you handle the data inside the tables. This is like filling your notebook with notes, editing them, or erasing some pages.

  • INSERT: Add new data into a table.
  • UPDATE: Change existing data.
  • DELETE: Remove data from a table.
  • MERGE: Combine data from two tables (used less often for beginners).

Example: Adding new students:

INSERT INTO Students VALUES (1, 'Sneha', '10A');
INSERT INTO Students VALUES (2, 'Rohan', '9B');
INSERT INTO Students VALUES (3, 'Priya', '10C');

Example: Updating a student’s class:

UPDATE Students
SET Class='10B'
WHERE Student_ID=1;

Example: Removing a student:

DELETE FROM Students
WHERE Student_ID=2;

3. Data Query Language (DQL)

DQL commands let you search and view data. The main command is SELECT. It’s like asking your notebook: "Show me all students in Class 10A" or "Give me the email of Rohan".

  • SELECT: View information from one or more tables.

Example: Show all students:

SELECT * FROM Students;

The * means “all columns”. If you only want the name and class of students, you can write: 

SELECT Name, Class FROM Students;

You can also filter data using WHERE: 

SELECT Name, Class FROM Students
WHERE Class='10A';

4. Data Control Language (DCL)

DCL commands control who can use the database. Think of it like giving friends permission to read your notebook, write in it, or take notes.

  • GRANT: Give permission to a user to access or change data.
  • REVOKE: Take back permission.

Example:

GRANT SELECT ON Students TO User1;  -- Allow user to see student data
REVOKE SELECT ON Students FROM User1;  -- Remove access

5. Transaction Control Language (TCL)

TCL commands help keep data correct and safe. Think of a transaction like making multiple edits in your notebook. You can save them all at once or undo them if something goes wrong.

  • COMMIT: Save all changes permanently.
  • ROLLBACK: Undo changes since the last save.
  • SAVEPOINT: Set a checkpoint to return to if needed.

Example:

SAVEPOINT Step1;  -- Create a checkpoint
UPDATE Students SET Class='10C' WHERE Student_ID=3; -- Make a change
ROLLBACK TO Step1;  -- Undo the change if needed
COMMIT;  -- Save the final changes

Example: Step by Step

Suppose we have a school database. Here is what we can do with SQL:

  • Create a table: 
    CREATE TABLE Students (Student_ID INT PRIMARY KEY, Name VARCHAR(50), Class VARCHAR(10));
  • Add students: 
    INSERT INTO Students VALUES (1, 'Sneha', '10A');
  • Update class: 
    UPDATE Students SET Class='10B' WHERE Student_ID=1;
  • View all students: 
    SELECT * FROM Students;
  • Delete a student: 
    DELETE FROM Students WHERE Student_ID=1;

Tips for Beginners

  • Start with CREATE, INSERT, SELECT, UPDATE, DELETE. These are the most common commands.
  • Practice on a small database first, like Students or Products.
  • Use SELECT often to check your data after inserting or updating.
  • Think of tables as sheets in your notebook, rows as entries, and columns as the type of information.
  • Always save your work using COMMIT if you are doing multiple updates.
  • Use ROLLBACK when you make mistakes, it’s like an undo button.
